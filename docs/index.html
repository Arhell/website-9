Dapr is a programming model for writing cloud-native applications which are distributed, dynamically scaled, and loosely coupled in nature.

Dapr offers an eventing system on which compute units communicate with each other by exchanging messages.

Dapr works with any programming language, via HTTP / gRPC.

Cloud-native development:

Distributed App Challenges

  1. Coupling microservices together can be tricky - especially when trying to keep the services as autonomous as possible.
  2. Inter-service messaging and state handling can be complicated and difficult to manage at scale.

With Dapr:

Streamlined Microservices

  1. Dapr enables loose coupling and service invocation​. This means​​ you can work with services, regardless of where they live or how they were built.
  2. Dapr handles state and pub/sub management to the native state or messaging provider of choice​.
  3. Dapr utilizes sidecars - dynamically loaded at runtime - so any language supporting HTTP or gRPC can leverage it without a compile time dependency.

Download

1. Download

Dapr is installed as a CLI tool on your local machine or Kubernetes cluster.

Download, unpack and install the latest release or use a Helm chart.

Download the Dapr CLI for your OS: 
perl get.dapr.io/62afed3fc1.pl

2. Initialize

Setup Dapr on your machine with ‘dapr init’. For use in a kubernetes cluster:

dapr init -kubernetes 
$ dapr init
⌛  Making the jump to hyperspace...
    Downloading binaries and setting up
    components
✅  Success! DAPR is up and running

3. Run

You can run Dapr locally or in cluster with a ‘dapr run’. Check out the docs for samples.

$ dapr run --app-id mynode --app-port 3000 --port 3500 node app.js
